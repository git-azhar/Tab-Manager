Overview

1. Introduction

This Privacy Policy describes how Tab Manager Pro ("the Extension"), developed by AZHAR (publisher: AZ&G), handles information when you install and use the extension in Google Chrome or Microsoft Edge.

Tab Manager Pro automatically manages inactive browser tabs according to user-defined rules. It can close, reload, mute, move, group, or snooze tabs based on inactivity periods and URL matching rules, and provides analytics and memory insights to help optimize browser performance. Version 4.0.0 adds bookmark management (import/export/favorites), browsing-history browsing with one-click reopen, on-demand tab grouping, session save/restore, a local heuristic site-safety indicator, and an optional PIN-protected site lock.

All of these features run locally. The Extension does not transmit your tabs, bookmarks, history, or any other browsing data to us or to any third party.

By installing or using the Extension, you agree to this Privacy Policy.

Data

2. What Data Is Collected

Data Type Collected? Details
Personal Identity Never Names, emails, phone numbers and addresses are never collected.
Passwords & Credentials Never Passwords, tokens and form data are never accessed.
Tab URLs Local Only URLs are processed locally to match rules and are never transmitted.
Analytics Local Only Tabs closed, warnings sent and statistics remain on your device.
Memory Statistics Local Only Tab memory information is processed locally.
IP Address / Location Never The extension never accesses your location or IP address.
Bookmarks Local Only Read and modified only on your device to display, add, import, and export bookmarks. Exports are written directly to a file you choose and are never uploaded.
Browsing History Local Only Queried locally so you can search and reopen recent pages. History is never transmitted off your device.
Site Lock PIN & Recovery Local Only Your PIN and optional recovery answer are never stored in plaintext — only one-way SHA-256 hashes are saved locally and used to verify unlock and reset attempts.
Saved Sessions Local Only Lists of tab URLs you explicitly save for later restore, stored on your device.
Safety Signals Local Only Site-safety labels are computed on-device using pattern heuristics. No URL is sent to any external service or reputation feed.
Permissions

3. Permissions Explained

  • storage Stores rules, analytics, schedules and settings locally.
  • tabs Allows monitoring of inactivity and performing configured actions.
  • notifications Displays warning and closure notifications.
  • alarms Schedules inactivity checks and pause windows.
  • idle Supports Idle-Only mode.
  • tabGroups Allows moving tabs into custom groups.
  • browsingData Enables cache, cookies and storage cleanup tools.
  • bookmarks Read, create, and remove bookmarks for the in-extension bookmark manager, import, and export.
  • history Search and display your recent browsing history so you can reopen or delete entries. Read locally; never transmitted.
  • favicon Displays site icons next to bookmarks and history entries.
  • scripting & host access Used to read per-tab JavaScript memory statistics and to display the site-lock screen on sites you have locked. No page content is collected or transmitted.
Storage

4. Data Storage & Retention

Extension data is stored on your device using chrome.storage.local and chrome.storage.sync. Synced data (rules, settings, locked-site list, PIN hash) may propagate to other devices where you are signed into the same browser profile; it is never sent to us.

The following information may be stored locally:

  • URL matching rules and configurations
  • Notification settings and pause schedules
  • Close history and analytics statistics
  • Memory monitoring information
  • Snoozed tab information
  • Saved sessions (tab URL lists you choose to save)
  • Locked-site list, a one-way SHA-256 hash of your PIN, and (if set) your recovery question plus a one-way hash of its answer — never the PIN or answer themselves
  • User preferences and extension settings

Rules, settings, and security configuration use chrome.storage.sync so they can follow your signed-in browser profile across devices; all other data uses chrome.storage.local. Bookmarks and history are managed through Chrome's own APIs and are not copied into extension storage. Session unlocks for locked sites are held only in memory and cleared when the browser restarts.
